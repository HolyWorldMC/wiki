# Руководство по написанию статей для википедии HolyWorld

## Общие принципы

### Целевая аудитория
Аудитория проекта, в том числе администраторы, ютуберы, новички и опытные игроки. Текст должен быть широким массам аудитории, но при этом оставаться информативным и полным.

### Стиль написания
- **Простой язык**: избегайте сложных терминов, пишите как обычный человек
- **Фактическая информация**: только факты, никаких советов и стратегий
- **Естественность**: текст должен читаться легко, без признаков автоматической генерации (ИИ)

## Структура статей

### Основной заголовок
```markdown
# Название механики
```

### Описание системы
Первый абзац должен содержать краткое описание того, что представляет собой механика. Используйте выделение **жирным** для названия системы.

**Пример:**
```markdown
**Скупщик** — это система продажи предметов за внутриигровую валюту (монеты) и очки скупщика.
```

### Доступ к системе
Если есть команды или способы доступа, указывайте их сразу после описания:

```markdown
Доступ к системе осуществляется через команду `/команда`.
```

## Оформление контента

### Заголовки разделов
Используйте заголовки второго уровня для основных разделов:
```markdown
## Основные функции
## Способы получения
## Типы предметов
```

### Заголовки подразделов
Для более детального разделения используйте третий уровень:
```markdown
### Категория предметов
### Особенности работы
```

### Таблицы
Предпочитайте таблицы длинным спискам. Таблицы должны быть информативными и легко читаемыми:

```markdown
| Предмет | Описание | Стоимость |
|---------|----------|-----------|
| **Название** | Краткое описание функции | цена |
```

### Списки
Используйте маркированные списки только для коротких перечислений:

```markdown
- Первый пункт
- Второй пункт
- Третий пункт
```

## Динамические значения

### Система ключей
Для значений, которые могут изменяться (цены, проценты, время), используйте специальные ключи:

```markdown
<!-- wiki[ключ-название] -->значение по умолчанию<!-- /wiki -->
```

**Примеры использования:**
- Цены: `<!-- wiki[buyer-coal-price] -->2<!-- /wiki --> монет`
- Проценты: `<!-- wiki[chance-drop] -->15<!-- /wiki -->% шанс`
- Время: `<!-- wiki[cooldown-time] -->60<!-- /wiki --> секунд`

### Именование ключей
- Используйте только латинские буквы, цифры и дефисы
- Формат: `категория-предмет-свойство`
- Примеры: `buyer-iron-price`, `region-damage-boost-xp`, `star-level-5`

## Стилистические правила

### Выделения текста
- **Жирный текст**: для названий предметов, команд, важных терминов
- *Курсив*: для примечаний и дополнительной информации
- `Код`: для команд и технических значений

### Описания команд
```markdown
**Команда:** `/команда параметр`
**Описание:** что делает команда
```

### Особенности и ограничения
Выделяйте важную информацию в отдельные абзацы или используйте специальное форматирование:

```markdown
**Важно:** критическая информация, которую нужно знать игроку.

**Ограничения:** список ограничений системы.
```

## Типичные разделы статей

### Для игровых механик
1. Описание системы
2. Доступ к системе  
3. Основные функции/типы
4. Способы получения/использования
5. Особенности и ограничения

### Для предметов
1. Описание предмета
2. Механика использования
3. Способы получения
4. Характеристики (в таблице)
5. Ограничения

### Для ивентов
1. Описание ивента
2. Механика проведения
3. Типы/варианты (если есть)
4. Система наград
5. Особенности участия

## Технические требования

### Кодировка и символы
- Используйте только стандартные символы UTF-8
- Избегайте невидимых символов и специальных Unicode-символов
- Проверяйте, что текст корректно отображается

### Ссылки
Используйте относительные ссылки на другие статьи:
```markdown
[название статьи](filename.md)
```

### Изображения
При необходимости размещайте изображения в папке `assets` соответствующей темы:
```markdown
![Описание изображения](assets/image.png)
```

## Что НЕ нужно делать

### Избегайте
- Советов и стратегий игры
- Сложных технических терминов без объяснения
- Длинных списков вместо таблиц
- Субъективных оценок ("лучший", "худший")
- Сложных AI-подобных структур

### Не используйте
- Структуры вида "# Экономические особенности: -... -... -..."
- Чрезмерно формальный язык
- Маркетинговые фразы
- Эмоциональные оценки

## Проверка качества

Перед отправкой статьи убедитесь:
- [ ] Текст понятен подростку 13 лет
- [ ] Нет советов, только факты
- [ ] Используются динамические ключи для изменяемых значений
- [ ] Таблицы оформлены корректно
- [ ] Нет грамматических ошибок
- [ ] Стиль соответствует существующим статьям

## Примеры хорошего оформления

### Описание предмета
```markdown
**Рюкзак** — это переносимый контейнер, функционирующий как шалкеровый ящик, который можно использовать без предварительной установки в мире.

## Размеры рюкзаков

| Тип | Цвет | Количество слотов | Рецепт крафта |
|-----|------|-------------------|---------------|
| **Начальный** | Розовый | 9 | Шалкер + 8 незеритовых слитков |
| **Средний** | Голубой | 15 | Розовый рюкзак + 8 незеритовых слитков |
```

### Описание механики
```markdown
**Биржа** — это торговая площадка для обмена внутриигровых монет на коины. Доступ к бирже осуществляется через команду `/exchange`.

## Принцип работы

Биржа функционирует как посредник между игроками, где одни продают монеты за коины, а другие покупают монеты, тратя коины.
```

Следуя этому руководству, вы создадите качественные статьи, которые будут полезны всем игрокам сервера.
